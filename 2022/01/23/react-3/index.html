严格模式

StrictMode 是一个用以标记出应用中存在潜在问题的工具，不会渲染任何真实的UI，它为其后代元素出发额外的检查和警告只在开发模式中运行，不会与生产模式冲突

好处

  • 识别不安全的生命周期组件
  • 有关旧式字符串ref用法的警告
  • 有关使用废弃的findDOMNODE方法的警告
  • 检测意外的副作用
  • 检测过时的context API
